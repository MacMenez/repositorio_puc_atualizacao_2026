# ETAPA DE USO DE ROTINA NO GITHUB

Quando apagar algo DE FORMA INCORRETA, entrar na pasta ou ir para a pasta principal da disciplina e usar o comando:
```git restore .```

0 Usar o Git Bash para tudo ou
* Git Bash: comandos do Git
* PS - Power Shell: Navegação de Diretório
* Prompt CMD: Compilação e Execução

> Para Depuração (_Debug_) usar os pontos laterais (_bullet point_) e usar a própria estrutura que o VS Code fornece. **ELE SÓ FUNCIONA COM OS PONTINHOS VERMELHOS MARCADOS**

1️⃣ entrar na branch  
```git checkout disciplina/algoritmosII```

2️⃣ criar arquivos e diretórios

3️⃣ testar/compilar/executar

4️⃣ salvar no git dentro da branch que desenvolvi a etapa 3

Conferir todas as alterações feitas:
```git status```

Se for subir tudo  
```git add .```  
```git commit -m "Mensagem"```  
```git push```

Se for subir arquivo por arquivo  
```git add pasta/arquivo```  
```git commit -m "Adicionando arquivo ..."```

5 voltar para main  
```git checkout main```

6 rodar o script na brach main  
```./scripts/puc atualizar algoritmosII```

7 fazer o pull request

8 criar outra disciplina/evento/monitoria/pesquisa como por exemplos  
```./scripts/puc criar disciplina algoritmosIII```  
```./scripts/puc criar pesquisa robotica```
```./scripts/puc criar eventos workshop-programacao```  
```./scripts/puc criar monitoria algoritmosII```
